LDAP und Samba tanzen gemeinsam

[@. Übungsziel]

Das Übungsziel ist es einen Verzeichnisserver und einen Dateiserver aufzusetzen. Dabei soll die Authentifizierung des Dateiservers über den Verzeichnisserver gehen. Als letztes soll eine Windows XP Maschine in die Domäne aufgenommen werden.


Du brauchst für diese Übung eine Fedora Maschine, welche der Datei- und Verzeichnisserver wird und einen Windows XP Client.

[A. Voraussetzungen]

Fedora

Folgende Pakete werden für die Übung benötigt. Alle außer die smbldap-tools hast du bereits in der letzten Übung installiert:

  • openldap-*.i386.rpm

  • openldap-clients-.*.i386.rpm

  • openldap-servers-.*.i386.rpm

  • nss_ldap-*.i386.rpm

  • smbldap-tools

Installiere die smbldap-tools mit yum.

[B. Konfiguration LDAP]

Stoppe den LDAP Server. Konfiguriere den LDAP Server wie in der Übung 3 beschrieben. Folgende sind die Unterschiede zum letzten LDAP Server:

  • dc=nwtk##,dc=local

  • Includiere folgendes zusätzliches Schema: /etc/openldap/schema/samba.schema

  • LDAP Adminisrator: cn=administrator,dc=nwtk##,dc=local

  • ACLS (Beschreibe die Bedeutung der ACLs in dein Protokoll) :access to attrs=userPassword,sambaLMPassword,sambaNTPassword

    • by self write

by dn="cn=administrator,dc=nwtk##,dc=local" write

by * authaccess to *

by dn="cn=administrator,dc=nwtk##,dc=local" write

by * read

Starte den Server und lege KEINE Einträge an. Passwort von LDAP Administrator nicht vergessen!

[C. Konfiguration Samba]

Hier wird Samba als primärer Domänencontroller konfiguriert. Die Domäne heißt nwtk##. Die gesamten SAM Accounts und Zugänge werden in OpenLDAP abgespeichert.


In unten stehender Konfiguration soll nun folgende Information eingetragen werden:

  • Der Windows Domänen Name lautet nwtk##

  • Der Server heißt SRV

  • Der Server erlaubt Roaming Profiles

  • Alle Freigaben befinden sich im Verzeichnis /home/samba, außer die Homeverzeichnisse selbst. Erzeuge alle notwendigen Verzeichnisse. Das Verzeichnis für die Profile muss die Berechtigung 2777 haben.

  • Alle mit XXX gefüllten Optionen müssen ausgefüllt werden

  • Samba muss das Passwort des LDAP Admins kennen. Dieser User istin der smb.conf angegeben. Das Passwort wird von Samba in der Datei /etc/samba/secrets.tbd abgespeichert. Das Passwort wird mit dem Befehl smbpasswd -w mysecretpwdeingegeben, wobei mysecretpwd das Password des LDAP Admins ist. Gib den Befehl mit dem korrekten Passwort ein.

  • Die Samba Konfigurationsdatei kann mit dem Befehl testparm getestet werden. Gib die Ausgabe von testparm in das Protokoll. Testparm sollte „Loaded services file OK.“ zurückliefern

  • Achte darauf, dass die Dienste nmb und smb gestartet sind. Wofür wird der Dienst nmb verwendet?

[D. Konfiguration smbldap-tools]

Die smbldap-tools vereinfachen stark die Verwaltung des Zusammenspiels zwischen Samba und OpenLDAP.

Die ersten Einträge in der ldap-Datenbank

LDAP muss die Standardeinträge der Windows Domäne erst erhalten. Dafür haben wir ja bisher das Verzeichnis nicht befüllt. Die Erstbefüllung mit allen wichtigen „OUs“ macht der Befehl

smbldap-populate

für uns.

Dafür muss zuersta llerdings smbldap konfiguriert werden. Es gibt dafür 2 Dateien:

  • /etc/smbldap-tools/smbldap.conf # Grundkonfiguration

  • /etc/smbldap-tools/smbldap_bind.conf # Zugangsdaten für die Anbindung an LDAP

/etc/smbldap-tools/smbldap.conf

Hier wird die Grundkonfiguration der smbldap-tools vorgenommen. Der wichtigste Eintrag ist hierbei eine neue SID. Diese kannst du mit dem Befehl

net getlocalsid

generieren. Dafür muss allerdings der Samba Server schon laufen. Trage das Ergebnis in die entsprechende Zeile der Konfigurationsdatei ein.

Weitere Einträge:

  • sambaDomain: Domänenname

  • slaveLDAP und masterLDAP: Die IP-Adresse des LDAP Servers (localhost)

  • ldapTLS=“0“, da wir kein TLS verwenden

  • suffix = eh klar

  • userSmbHome=“\\<Samba-Rechnername-nicht-Domäne>\%U“

  • userProfile=“\\<Samba-Rechnername-nicht-Domäne>\profiles\%U

/etc/smbldap-tools/smbldap_bind.conf

Hier wird die Authentifizierung gegen den LDAP-Server konfiguriert. Einfach den LDAP Admin mit dem gesamten DN angeben und das entsprechende Passwort. Bei slave und master angeben.


Nun kann der Befehl smbldap-populate aufgerufen werden. Es sollen so einige Einträge erzeugt werden. Am Ende das Passwort von root eingeben. Am besten das selbe wie vom ldap Administrator.

Wenn dem so ist bitte jubeln!! Das war schon ein großer Brocken.

smbldap-tools, die Skripte

Es gibt von smbldap-tools Skripte, mit denen man gleichzeitig für Ldap, Samba und dem Linuxsystem User verwalten kann. Diese Skripte heißen gleich wie die Befehle unter Linux, allerdings mit smbldap- Vorangestellt. Beispiel: smbldap-useradd

[E. Zusammenspiel mit Windows]

Damit wir einen Rechner in die Domäne aufnehmen können, brauchen wir dazu noch Benutzer. Einen

  • Domänenadmin, zum hinzufügen des Windows Rechners an die Domäne (gibt es zwar schon root, aber wir fügen der Sauberkeit halber noch einen hinzu) und einen

  • normalen User, zum Arbeiten am Windows Client

Vorher müssen wir aber dem Linuxsystem noch sagen, dass User und Gruppen nicht nur in den Dateien (/etc/passwd, usw.) sondern auch im LDAP suchen soll. Dafür müssen wir in der Datei /etc/nsswitch.conf die entsprechenden Zeilen ändern auf:

passwd: files ldap

shadow: files ldap

group: files ldap

hosts: files dns wins

Damit sollten die Befehle

getent group bzw. getent passwd

nun auch die Gruppen bzw. die User aus dem ldap anzeigen. Teste dies.

Hinzufügen des Domänenadmins

Wir legen nun einen Domänenadmin mit dem Namen „anmelder“ an. Gib dafür den Befehl

smbldap-useradd -m -a anmelder und anschließend

smbldap-passwd anmelder ein.

Anschließend wird der User noch zu den Domain Admins hinzugefügt

smbldap-usermod -G “Domain Admins“ anmelder

Wir müssen noch der Gruppe „Domain Admins“ die Rechte geben, Maschinen in die Domäne aufzunehmen

net -U roo%Passwort rpc rights grant 'Domain Admins' SeMachineAccountPrivilege


Wer sich über den Befehl und der Syntax aufregt: Das ist KEIN Linuxbefehl!!
